## 实测解读示例:亚洲老挝爬奔昆南 1km² 矿权(冬季 DEM)

> 用一份真实分析结果把"参数→图像→地质含义"串起来。
> run:`results/亚洲老挝爬奔昆南1km2矿权_1780478504/structural/20260604_182109_struct_0000/`。
> 数值均为该 run 各 GeoTIFF/GeoJSON 实测统计。

**输入与地形概况**:AOI ≈ **1.40×1.30 km(1.81 km²)**;DEM **87×100** 像元、像元 ≈ **14 m**、EPSG:4326;
高程 **567–891 m,高差 324 m**;坡度 **均值 26.5°、最大 54°、57% 面积 ≥25°** → 强切割中-高山区。
→ 这解释了为何此处要**封顶坡度门控**:全区普遍陡,若按坡度高分位门控会把真实断裂边缘也滤掉。

| 图像/产物 | 本例实测 | 解读 |
|---|---|---|
| **图2-1A 山体阴影+断裂** | 多方位阴影上提取 **6 条线性体**(红线叠加) | 见下方走向分组;线性体沿沟谷/坡折展布,反映构造-侵蚀格架 |
| **构造方向(玫瑰图/统计)** | 走向 **25°、25°、35°、36°、109°、115°**;长 272–731 m | **两组**:主导 **NNE–NE(≈25–36°,4 条)** + 次级 **ESE(≈109–115°,2 条)**,近共轭。NE 为区域主构造格架,ESE 为横切组,**两组交汇处为找矿优先靶区** |
| **distance_to_lineament** | 0–350 m,**均值仅 98 m,64% 面积在断裂 100 m 内** | 断裂网络**密集**;就构造控矿先验而言,本区绝大部分都属"近断裂高权重"带 |
| **lineament_density** | 峰值 0.038 | 密度热点对应线性体交汇/并行带 |
| **SVF 天空视域** | 0.456–1.000,均值 0.743 | 谷底显著低(0.46,深切沟谷)、脊部接近 1(开阔);清晰区分脊/谷,**比单方位 2-1A 更稳健地勾出沟谷线** |
| **openness 开度** | 52.2–108.3°,均值 77° | 与 SVF 同向:低值=深谷/凹地,高值=脊/凸起 |
| **curvature 曲率** | −77 ~ +11;**谷(负)54% / 脊(正)46%** | 正负高频相间 → 强烈脊谷交替,典型构造-侵蚀地貌;谷线(强负)多与 NE 线性体重合,佐证沟谷受构造控制 |
| **图2-1C 地形渲染** | 纯地形暖色调(该 run 未叠 Landsat) | 宏观见 NE 向脊谷相间的"梳状"地貌;若该项目冬季有 Landsat 8,则自动用 SWIR 7-6-4+IHS 融合显岩性 |

**综合判读**:本区为**强切割中-高山**,发育 **NE 主、ESE 次两组线性构造**,断裂网络密集(64% 面积近断裂)。
找矿上应优先关注 **NE×ESE 交汇结点**;叠加 geo-analyser 蚀变、geo-insar 形变后可收敛靶区。
**分辨率局限**:1 km²×14 m DEM 仅 87×100 像元,提取的 6 条为**地形显著**线性体,细小/隐伏构造受分辨率限制,
结论属决策支持,需更高分辨率 DEM 或野外核实。

> 对比另一极端——**山东招远(缓坡小区)**:坡度多 <5°,固定门控会"零提取",靠**门控下探**才得 6 条(主向 5°/95°)。
> 两例共同说明:坡度门控必须**双向自适应+封顶**,才能在陡区与缓区都不丢真实构造。

## 1. 图2-1A 山体阴影遥感地质构造解译图

### 参数
- **山体阴影算法**:Horn 3×3 梯度 + 标准照度模型(`terrain_utils.py:62-87`)。
  - 方位角 `azimuth`:默认 **315°**(西北光照,UI 可调,`structural_engine.py:169`)。
  - 高度角 `altitude`:**30°**(UI 默认,`app.py` 传入;函数签名默认 45°)。
  - 垂直夸张 `z_factor`:**制图用 5×**(增强地形可读性,`structural_engine.py` z_display=5),
    **落盘 tif 用 1×**(保留真实坡度量值)。
  - 梯度前高斯平滑 `SMOOTH_SIGMA=1.0` 像元(抑制 DEM 量化阶梯,`terrain_utils.py:21`)。
  - 输出灰度 uint8 0–255,显示 `vmin=0, vmax=255`。
- **叠加要素**:
  - **自动提取线性体(红线)**:`linewidth=1.2, alpha=0.85`(`structural_map_viz.py:_add_lineaments`)——
    这是图面唯一"真实绘制"的构造要素,图例同步标注"自动提取线性体(断裂)"。
  - **等高线**:对 DEM 先高斯 `sigma=1.5` 平滑,自动取 `n_levels≈12`、步长取 1/2/2.5/5×10ⁿ 的"美观值",
    标注 `%d m`(`structural_map_viz.py:_add_contours`)。
  - **工作区边界**:红色多边形 + A/B/C/D 角标。
  - **坐标网格**:间隔 **1/12°(=5′)**,标注 度°分.分′ N/E(`_add_coordinate_grid`)。
  - **比例尺 5 km**、**指北针**、**构造图例**。

### 物理/地质含义与判读
山体阴影把 DEM 的微地形起伏以明暗表达:**线状的明暗突变带 = 断裂/线性构造**(两侧错位致地表高差突变),
平行条纹 = 节理系,圆形暗影边界 = 环形构造。**判读重点是红色线性体的展布、密度与方向集中度**。

### 局限
- 单方位 315° 光照对**走向≈NW-SE(与光照平行)**的线性体不敏感(方位盲区)——这正是另产出 SVF/开度(见 §5)与
  多方位提取(线性体本身用多方位基底,§6)的原因。
- z=5 夸张是视觉手段,勿据 PNG 量取坡度;定量请用 `slope.tif`。

## 2. 图2-1B 坡向分析遥感地质构造解译图

### 参数(HSV 合成,`structural_map_viz.py:109-135`)
- **色相 H = 坡向 / 360**(罗盘色盘:N=青、E=绿、S=红、W=品红)。
- **饱和度 S = 0.25 + 0.75 × clip(slope/12°, 0, 1)**;平坦区(aspect=-1)S=0(去色变灰)——缓坡浅色、陡坡(≥12°)满饱和。
- **明度 V = 0.4 + 0.6 × (hillshade/255)**(山体阴影驱动立体感,压到 [0.4,1] 避免阴影区丢色相)。
- 右上角**指南针色盘图例**(N 上、顺时针,`_add_aspect_colorwheel`)。

### 含义与判读
坡向反映坡面朝向,间接指示节理产状/褶皱翼向/构造走向。**同一色调成片或线状定向排列**提示统一的构造方向;
通过色盘读优势朝向 → 推断区域构造应力/走向。

### 局限(重要)
坡向是**坡面朝向**,只是构造方向的**间接代理**,受水系/地形支配。把坡向色直接当"节理/褶皱方向"会**高估**其含义——
更直接的二阶依据见 `curvature.tif`(§5)。

## 3. 图2-1C 地形渲染遥感地质构造解译图

### 参数(`terrain_utils.py:compute_terrain_render`)
- **多方位山体阴影融合**:方位角 (315,45,135,225)°、权重 (0.3,0.3,0.2,0.2)、z=1(`compute_multidirectional_hillshade`)——四象限消阴影伪迹。
- **坡度着色**:`slope_norm = clip(slope/30°,0,1)`,`slope_shade = 1 − 0.5×slope_norm`;`intensity = clip(combined_shade × slope_shade,0,1)`。
- **影像融合(IHS)**:若有 Landsat,转 HSV(IHS 代理),**用地形 intensity 替换明度 V = 0.15 + 0.85×intensity**,保留岩性色相/饱和度(`terrain_utils.py:265-271`)。无影像时走暖色调纯地形渲染。
- **Landsat 波段组合**:优先 **SWIR 地质组合 7-6-4**(SWIR2/SWIR1/Red,突出岩性/蚀变),缺 SWIR 回退真彩色 4-3-2;各波段 2% 百分位拉伸(`structural_engine.py` Landsat 加载段、`percent_stretch`)。

### 含义与判读
三维地貌 + 岩性色彩综合视图:明亮高地=隆起/背斜,深暗低地=沉降/向斜,线状深沟=断层控制谷地;
SWIR 色彩差异辅助**岩性边界**识别。判读宏观构造格架(盆-山、隆-坳)与岩性分区。

### 局限
IHS 融合保留色相但明度被地形主导,**不可据此做定量光谱分析**;SWIR 组合需影像确实含 B6/B7。

## 4. rose_diagram.png 构造方向玫瑰图

### 参数(`lineament.py:plot_rose_diagram`)
- **按线性体长度加权**的走向直方图;36 个扇区(`nb=36`);0–180° 统计后**双向镜像**到 0–360°。
- 极坐标:N 上(`set_theta_zero_location('N')`)、顺时针(`set_theta_direction(-1)`);`dpi=150`。

### 含义与判读
花瓣越长 = 该走向的断裂总长越大 = **优势构造方向**。多组花瓣 = 多期/多组构造;与 `metadata.structural_stats.dominant_strikes_deg`(取前 3 峰)一致。

### 局限
仅反映**地形可识别**的线性体方向;短小/隐伏构造可能漏统计。线性体少时(如小而平缓 AOI)玫瑰图不稳定。

## 5. 地形量算栅格 + 光照无关增强 + 曲率

### 5.1 hillshade_315.tif / aspect.tif / slope.tif
- `hillshade_315.tif`:**z=1 无夸张**版本(与 2-1A 的 z=5 制图版不同),uint8 0–255。
- `aspect.tif`:0–360°,**−1=平坦**(判据 `slope<1.5°`,`terrain_utils.py:104`),float32。
- `slope.tif`:0–90° 真实坡度角,float32。**定量坡度以此为准**。

### 5.2 svf.tif 天空视域因子 + openness.tif 地形开度(`compute_skyview_openness`)
- **算法**:地平线角扫描,方向数 `n_dir=16`、半径 `max_radius_px=12` 像元;纯 numpy(不依赖 rvt-py)。
- **SVF = 1 − mean_dir( sin(max(0, 地平线角)) )**,值域 0–1:**谷/凹陷低、脊/开阔高**。
- **正开度 = mean_dir(90° − 地平线角)**(度):**脊/凸起大、谷小**。
- **关键价值**:二者**与光源方向无关**,无 2-1A 的方位盲区,揭示断裂/线性构造更稳健。

### 5.3 curvature.tif 地形曲率(`compute_curvature`)
- **总曲率 ∝ −(z_xx + z_yy)**(平滑后有限差分拉普拉斯,按 x/y 米尺度归一,×100 提升可读性)。
- 符号:**脊线/背斜枢纽为正(凸),谷线/向斜为负(凹)**。
- 用途:为褶皱/脊谷判读提供比"坡向上色"更直接的**二阶地貌依据**。

### 局限
SVF/开度的半径 12 像元 = 只感知**局地**地平线(随像元尺寸而定的物理半径);曲率对 DEM 噪声敏感,已用 σ≥1 平滑缓解。

## 6. 线性体衍生产物(下游核心)+ 提取参数详解

### 6.1 提取流程参数(`lineament.py:extract_lineaments`)
1. **基底**:多方位融合山体阴影(光照无关),2–98% 百分位拉伸到 [0,1]。
2. **边缘**:Canny,`sigma=1.5`,**分位阈值** `use_quantiles=True, low=0.8, high=0.92`(适配任意动态范围,解决"阴影对比度过窄检不出"问题)。
3. **坡度门控(自适应)**:阈值 = **max(1.0°, 坡度第 65 百分位)**(`slope_gate_deg=1.0, slope_gate_pct=65`)——
   陡区只留强边缘、缓区(小而平 AOI)自动放宽,避免固定阈值一刀切。
4. **细化**:`skeletonize`。
5. **提线**:概率霍夫,投票阈值 = **clip(min(H,W)//4, 5, 10)**(小图降阈)、`line_length=min_length/像元`、`line_gap=3`。
6. **长度过滤**:`min_length_m=300`(短于此剔除,抑制水系/噪声)。

### 6.2 产物
- **distance_to_lineament.tif(米)**:`distance_transform_edt`,`sampling=(像元_y,像元_x)`→真实米距;**近断裂=0**。
  *下游杠杆最大*:analyser 构造加权、exploration 构造控矿因子、insar 形变归因都基于它。
- **lineament_density.tif(0–1)**:对线性体二值掩膜做 `uniform_filter`,窗口 = `density_window_m=1000m`/像元 → 窗内线性体像元占比。
- **lineaments.geojson**:LineString;属性 `strike_deg`(走向,0–180°,正北为0顺时针)、`length_m`。
- **structural_stats(metadata.json)**:`n_lineaments`、`total_lineament_length_km`、`lineament_density_mean`、`dominant_strikes_deg`(前3峰)、`elevation_range_m`。

### 判读与分析
**成矿模型 源-运-储**中,断裂是"运"(导矿/容矿通道)。判读要点:
- **距断裂近 + 蚀变异常叠合** = 高优先靶区;
- **断裂交汇/密度高值区**(密度图热点)= 经典控矿位;
- **优势走向**(玫瑰图)= 区域构造格架与控矿方向。

### 局限(诚实边界)
自动提取**天然含噪**:水系、山脊、地貌陡坎都可能冒充断裂。已用坡度门控 + 长度过滤抑制,但结果是**决策支持层,
非已验证构造地质结论**,断裂位置/性质需野外或多源核实。小而平缓 AOI(如 2–3 km²、相对高差几十米)可能仅提取到 1–2 条甚至 0 条。

## 9. 整体判读建议(把图件串起来用)

1. **构造格架**:2-1C(宏观盆-山/隆-坳)→ 2-1A 红线 + SVF/开度(断裂定位)→ 玫瑰图(优势走向)。
2. **褶皱/脊谷**:curvature(正脊负谷)+ 2-1B 坡向(翼向)互证,勿单凭坡向色。
3. **靶区**:distance_to_lineament + density 热点 → 叠加蚀变(geo-analyser)/形变(geo-insar)→ 收敛找矿/活动断层靶区。
4. **定量**:一律用 GeoTIFF(slope/distance/density/curvature),PNG 仅供视觉判读。
5. **可信度**:所有自动产物为**决策支持**,关键结论需野外/多源核实;小而平缓区结果不稳定时,降低 `min_length_m`/坡度门控可提灵敏度,但会引入更多噪声,需权衡。

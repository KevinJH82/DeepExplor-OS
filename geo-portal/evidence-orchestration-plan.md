# 证据二级编排引擎方案

## Summary

当前一个主编排引擎不够。它适合生成项目级流程，但不适合持续处理“数据完成后证据如何分配、失败后如何重跑、缺失证据如何降级、证据权重如何随矿种/数据质量变化”的动态问题。

新增一个证据编排引擎：数据阶段完成后自动生成证据分配方案，用户可审核、修改、重跑，并由该方案驱动 2D 证据层、3D 融合和证据链叙事。

## Key Changes

- 保留现有 `orchestrator` 作为主编排：负责 `plan -> data -> evidence -> model3d -> drill -> report` 的阶段级计划。
- 新增 BFF 内置 evidence planner：
  - 输入：项目矿种、AOI、已下载数据源、已有历史产物、各证据服务能力、当前证据状态。
  - 输出：证据任务清单、推荐权重、依赖关系、运行优先级、重跑/降级策略、进入 3D 的门控条件。
- 数据阶段完成后自动触发证据编排，前端展示“证据编排单”：
  - 用户可勾选/取消证据任务。
  - 用户可调整权重。
  - 用户可强制重跑、跳过、降级某项证据。
  - 系统明确说明为什么推荐或不推荐某项证据。
- 证据阶段不再只靠前端按钮状态判断是否可进入 3D，而是使用证据编排结果：
  - 达到最低证据数量。
  - 关键证据是否满足矿种模型要求。
  - 缺失证据是否已有降级解释。
  - 已完成证据是否有可用图层、摘要或模型输入记录。

## Public Interfaces

- `POST /api/runs/{trace_id}/evidence-plan`
  - 生成或刷新证据编排单。
- `PATCH /api/runs/{trace_id}/evidence-plan`
  - 保存用户审核后的证据选择、权重、重跑策略。
- `POST /api/runs/{trace_id}/evidence-plan/execute`
  - 按证据编排单进入执行状态，前端按返回任务启动具体证据服务。

## Frontend Behavior

- 数据阶段完成后，自动进入“证据编排审核”状态，而不是直接卡在证据层。
- 右侧面板从“证据矩阵”升级为“证据编排单 + 证据矩阵”：
  - 上半部分展示系统推荐：为什么跑蚀变、构造、物探、化探、形变。
  - 中部展示可编辑权重和参与状态。
  - 下方提供明确按钮：`执行证据编排`、`继续运行待执行证据`、`重新生成证据编排`。
- 中央 AOI 不再只显示“未运行”，而是显示当前证据编排状态。
- 证据链叙事引用证据编排单中的 `rationale`，说明每项证据为什么被选中、为什么被降级、为什么可以进入 3D。

## Test Plan

- 新建金属矿项目，完成数据阶段后应自动生成证据编排单。
- 新建油气项目，证据推荐应区别于金属矿：更强调构造、盆地/物探、形变和地球化学约束，蚀变不默认作为核心证据。
- 部分证据完成、部分待运行时，页面必须提供继续执行入口，不能出现死锁。
- 某项证据失败时，用户应能选择重试、跳过或降级，并且证据链说明必须同步变化。
- 用户调整权重后，3D 融合输入和证据链叙事应使用调整后的权重。
- 刷新页面后，证据编排单、证据状态、用户修改和运行结果必须恢复。

## Assumptions

- 采用二级编排引擎架构。
- 数据完成后自动触发证据编排。
- 用户采用“审核可改”模式：系统推荐，用户可干预。
- 第一版先做 BFF 内部 evidence planner；后续如果逻辑复杂，再拆成独立服务。

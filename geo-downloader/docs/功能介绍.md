# DeepSearch Downloader — 功能介绍

DeepSearch Downloader 是一款面向地质勘探场景的卫星遥感数据自动下载与处理工具，支持 **33 个主流卫星传感器**，提供 Web 界面与命令行两种使用方式，全流程覆盖数据搜索、下载、裁剪、衍生产品计算及标准交付打包。

**最后更新：** 2026-04-15  
**新增功能：** ASTER L1T、EnMAP 自动化下载（CAS SSO + Playwright + FTPS）、矿物波段比值分析、高分辨率遥感支持、多区域并发处理

## 二、核心功能

### 1. 区域驱动的批量下载

以 **KML / ovKML / KMZ 文件**定义研究区域，支持：
- 单个区域文件（`.kml` / `.ovkml` / `.kmz` / `.ovkmz`）
- 文件夹批量处理（目录下所有 KML 文件逐一处理）
- 多边形区域与点位区域（点位自动缓冲 ~11km 范围）
- **新增：** Excel 坐标文件上传（`.xlsx` / `.xls`，自动识别经纬度列）

一次操作可同时对一个区域下载多个传感器数据，传感器并发执行（最多 4 个同时运行），最大化下载效率。**新增：** 多区域文件夹批量模式支持多个区域同时并发处理（默认 2 个区域并行，可通过 `--area-workers` 调整）。

### 2. 多线程加速下载

对支持 HTTP Range 请求的数据源（DEM、SRTM、OPERA、ECOSTRESS、Sentinel-2 等），自动启用多线程分块下载：
- 默认 8 线程并行，每块 1MB，小于 1MB 的文件退化为单线程
- 单文件下载速度显著提升，充分利用带宽
- **新增：** EnMAP 全自动化下载（CAS SSO 认证 + Playwright 无头浏览器 + 订单轮询），无需手动操作
- **连接挂死保护**：iter_content 超过 120 秒无新数据时主动断开并触发重试，彻底消除代理半开连接导致的永久挂死问题

### 3. 断点续传与网络重试

所有下载器支持完整的中断恢复机制：
- **文件级断点续传**：中断后重启，已完成文件自动跳过，`.part` 临时文件从断点位置继续
- **网络自动重试**：断网或超时时自动重试，最多 10 次，指数退避（最长等待 120 秒）
- **SSE 重连**：页面刷新或网络中断后，自动恢复日志流，实时历史日志完整回放
- **传感器级重试**：单个传感器失败自动重试，不影响其他传感器执行
- **EnMAP 订单超时**：下单后最长轮询 8 小时（可通过 `--order-timeout` 调整），订单未完成时跳过打包，等下次运行自动恢复

### 4. 自动裁剪

下载完成后，自动将影像裁剪至 KML 多边形范围内：
- 支持 GeoTIFF、JP2、IMG、NetCDF 等格式
- 支持 Sentinel-2 ZIP 产品自动解压后裁剪
- 裁剪结果添加 `_clipped` 后缀，并删除原始全景文件以节省存储
- **新增：** 支持自定义裁剪分辨率（重采样到指定像元大小）

### 5. 衍生产品计算

根据已下载的影像自动计算以下遥感衍生产品：

| 产品 | 数据来源 | 用途 | 新增？ |
|------|----------|------|--------|
| 地表温度（LST） | Landsat Band10 / ASTER 热红外 | 地热异常、热液蚀变探测 | ✓ 已有 |
| 温度梯度 | 基于 LST，Sobel 算子 | 构造断裂带识别 | ✓ 已有 |
| 温度异常梯度 | 局部背景差值后求梯度 | 热液/地热异常精细定位 | ✓ 已有 |
| OTCI 植被指数 | Sentinel-2 红边波段 | 植被覆盖评估，地表扰动监测 | ✓ 已有 |
| **矿物波段比值** | **ASTER L2 SWIR 波段** | **高岭石、明矾石、白云母填图** | **✨ 新增（2026）** |

**矿物波段比值分析（新增功能）：**
- RBD (Relative Band Depth) 2.2µm 吸收深度指数：`(B5+B7)/(2×B6)`
- Kaolin_Ratio 简单比值：`B5/B6`
- Alunite_vs_Kaolin 区分指数：`B7/B6`
- Clay_Broad 广义粘土矿物指数：`(B5+B7)/(B6+B8)`

自动输出 GeoTIFF 格式，支持 QGIS/ArcGIS 加载分析，同时生成叠加可视化 PNG。

### 6. 标准交付目录打包

将下载和处理结果按照标准文件架构整理为可直接交付的目录：

```

├── {区域名}.kml
├── {区域名}_卫星数据下载报告_YYYYMMDD.docx   ← 自动生成
├── data-矿权-夏季（6-8月）/
│   ├── Sentinel 2 L2/       B01–B12, B8A（低分波段已提升至 10m）
│   ├── Landsat 8 L2/        B1–B11（重采样至 15m）
│   ├── Landsat 7 ETM+/      B1–B8（多光谱重采样至 15m，全色 B8 保持原始 15m）
│   ├── ASTER L1T/           B1–B14（新传感器，原始 15m）
│   ├── ASTER L2/            B1–B14（含热红外，原始 15–90m）
│   ├── EMIT L2A/            SPECTRAL_IMAGE.tif（重采样至 30m）
│   ├── DEM.tif              （重采样至 15m）
│   ├── 地表温度.tif
│   ├── 温度梯度.tif
│   ├── 温度异常梯度.tif
│   ├── OTCI.tif
│   ├── 高岭石_RBD_2200.tif  （新增产品）
```

### 7. 交付图像分辨率提升

打包阶段对低分辨率波段自动进行双线性重采样，提升交付图像的像素密度，改善 GIS 叠合与视觉判读效果：

| 传感器 | 原始分辨率 | 交付分辨率 | 说明 |
|--------|-----------|-----------|------|
| Sentinel-2 B02/B03/B04/B08 | 10m | 10m | 原始分辨率，不变 |
| Sentinel-2 B01/B05–B09/B8A | 20–60m | 10m | 与高分波段对齐 |
| Landsat 8/9 全波段 | 30m | 15m | 双线性插值 |
| Landsat 7 多光谱 B1–B7 | 30m | 15m | 双线性插值 |
| Landsat 7 全色 B8 | 15m | 15m | 原始分辨率，不变 |
| ASTER L1T 全波段 | 15m | 15m | 原始分辨率，不变 |
| ASTER L2 VN 波段 | 15m | 15m | 原始分辨率，不变 |
| ASTER L2 SWIR 波段 | 30m | 30m | 原始分辨率，不变 |
| EMIT 高光谱 | 60m | 30m | 双线性插值 |
| DEM（Copernicus GLO-30） | 30m | 15m | 双线性插值 |

> 重采样不增加实际信息量，仅提升像素密度与显示效果，地理坐标精度不变。双线性插值用于光学波段，近邻法用于分类波段。

### 8. 自动生成下载报告

每次打包完成后，自动在 `delivery/{区域名}/` 目录生成 Word 格式下载报告（需安装 `python-docx`，未安装时跳过）：

| 报告章节 | 内容 |
|----------|------|
| 一、任务概览 | 任务时间、耗时、搜索日期范围、文件数、传感器数、交付包大小 |
| 二、各传感器下载明细 | 传感器名称、类型、文件数、交付分辨率 |
| 三、未获取数据传感器 | 下载失败或无数据的传感器及原因 |
| 四、交付成果 | 夏/冬季包内容描述、总大小、输出路径 |
| 五、分辨率说明 | 本次重采样提升的详细说明 |
| **六、衍生产品说明**（新增） | 地表温度、矿物指数等衍生产品的物理意义与解读指南 |

报告文件名格式：`{区域名}_卫星数据下载报告_{日期}.docx`

### 9. 实时进度追踪

Web 界面提供全程可视化进度反馈：

- **整体进度条**：显示当前处理传感器、已下载景数、剩余时间估算
- **单文件进度条**：实时显示当前文件下载百分比（基于 tqdm 解析）
- **三阶段追踪**：下载 → 衍生产品计算 → 打包交付，各阶段独立显示
- **彩色日志**：成功（绿）、警告（黄）、错误（红）实时滚动，页面刷新后日志完整恢复
- **多任务并发**：支持最多 3 个传感器同时下载 + 最多 2 个打包任务并发，任务列表显示所有执行状态

### 10. 代理自动检测

启动下载时自动探测本地代理端口，发现可用端口后自动为下载子进程注入代理配置，无需手动设置环境变量，方便在网络受限环境下使用。

**探测顺序：**
1. 快柠檬 SOCKS5：端口 10793（优先）
2. 快柠檬 HTTP：端口 10792（回退）
3. Clash / 通用代理：依次尝试 7890 / 1087 / 8080

**智能直连（no_proxy）：** Copernicus、DLR EOWEB、NASA Earthdata、USGS 等欧洲和 NASA 服务默认直连，不经过代理，避免因代理节点限速而影响这些原本直连更快的数据源。

**新增：** 支持手动代理配置和直连模式切换，可为不同传感器配置不同代理策略（如 Copernicus 直连，其他走代理）。

### 11. 传感器预评估

下载前可一键预检各传感器在目标区域和时间范围内的数据可用性，快速筛选有效数据源，避免无效等待。

返回结果包括：
- 各传感器的可用景数
- 云量覆盖率统计
- 预计下载时间和文件大小
- 数据质量评分

### 12. 配置持久化

Web 界面中填写的账号信息、传感器选择、日期、云量等所有参数均可一键保存，下次打开自动加载。

**新增：** 
- 任务历史管理：保存过往任务配置，一键快速重新下载
- 任务复制：基于已完成的任务快速生成新任务
- 配置导出/导入：支持将配置导出为 JSON 文件，跨机器共享

### 13. GEE 自定义集合支持

支持通过 Google Earth Engine 的自定义资产集合（Custom Collections）下载任意栅格数据：

- 支持 S2、Landsat、MODIS 等官方集合
- 支持用户自定义算法计算的结果（如 NDVI、LST 等）
- 自动分块下载大区域数据
- 服务账号认证，支持离线自动化下载

### 14. 下载成功率统计与智能排序

系统自动记录每个传感器历史下载的成功与失败次数，在多传感器任务中按成功率降序排列执行顺序，让成功率高的传感器优先运行，尽早完成可靠下载，降低整体任务等待时间。

## 三、传感器模式

### 交付模式（默认）

自动勾选满足标准交付架构所需的核心传感器组合：
- 高程：SRTM
- 光学：Sentinel-2 + Landsat 8/9 + Landsat 7 + ASTER L2
- 高光谱：EMIT L2A
- 热红外：ECOSTRESS
- 衍生产品：所有地表温度与矿物指数

适合快速生成标准勘探交付包。

### 自定义模式

手动点选任意传感器，完全自由搭配。适合特殊应用场景或数据融合实验。

## 四、使用场景示例

### 场景 1：矿产勘探
- **选择传感器**：SRTM + Sentinel-2 + Landsat 8 + ASTER L2 + EMIT + 矿物波段比值
- **衍生产品**：矿物填图（高岭石、绿泥石、明矾石等）+ 地表温度异常
- **应用**：圈定矿化蚀变带

### 场景 2：地热勘探
- **选择传感器**：SRTM + Landsat 8 + ASTER L2 + ECOSTRESS
- **衍生产品**：地表温度 + 温度异常梯度
- **应用**：定位地热异常区，指导钻探位置

### 场景 3：断裂带识别
- **选择传感器**：SRTM + Landsat 8 + ALOS-2 PALSAR-2 + SAR 干涉相位
- **衍生产品**：DEM 流向分析 + SAR 相干性
- **应用**：识别隐伏断裂，评估地震风险

### 场景 4：植被覆盖评估
- **选择传感器**：Sentinel-2 + MODIS
- **衍生产品**：NDVI 时序分析，OTCI 植被指数
- **应用**：监测生态恢复状况，评估草地退化

# DeepSearch Downloader — 操作指南

---

## 一、首次使用准备

### 1.1 注册账号

根据需要下载的传感器，提前注册对应账号：

| 账号 | 注册地址 | 所需传感器 |
|------|----------|----------|
| NASA Earthdata（免费） | https://urs.earthdata.nasa.gov/ | Sentinel-1、OPERA、ALOS、ALOS-2、EMIT、AVIRIS-NG、DESIS、Hyperion、ASTER L1T/L2、MODIS、GEDI、ECOSTRESS |
| Copernicus（免费） | https://dataspace.copernicus.eu/ | Sentinel-2 |
| ASI PRISMA（免费） | https://prisma.asi.it/ | PRISMA |
| DLR EOWEB（免费） | https://eoweb.dlr.de/ | EnMAP、DESIS |

> DEM、SRTM、Landsat 8/9、Landsat 7、Landsat TIRS 无需账号，可直接下载。

### 1.2 安装依赖

```bash
cd geo-downloader
pip install -r requirements.txt
```

`requirements.txt` 中包含以下关键依赖：
- `python-docx`：自动生成 Word 格式下载报告
- `playwright`：EnMAP 全自动化下载所需（CAS SSO + 浏览器操作）

若已有旧版依赖环境，单独补装：

```bash
pip install python-docx playwright
playwright install chromium   # EnMAP 下载所需，约 150 MB
```

> 不使用 EnMAP 传感器时，可跳过 Playwright 安装。

### 1.3 启动服务

```bash
python3 web/app.py
```

浏览器访问 **http://localhost:8080**

## 二、界面总览

页面分为三个区域：
- **左栏**：账号配置（各平台用户名/密码/API Key）
- **右栏**：任务配置（KML 区域、传感器选择、时间范围、参数）
- **底部**：整体进度条 + 实时运行日志

顶部操作栏包含：**保存配置**、**数据预检**、**开始下载** 三个主要按钮。

## 三、配置账号

在左栏填写对应平台的账号信息：

- **Copernicus**：Sentinel-2 下载所需
- **NASA Earthdata**：Sentinel-1、OPERA、ALOS、ALOS-2、EMIT、AVIRIS-NG、DESIS、Hyperion、ASTER L1T/L2、MODIS、GEDI、ECOSTRESS 下载所需
- **DLR EOWEB**：EnMAP、DESIS 下载所需
- **ASI PRISMA**：PRISMA 下载所需

填写完成后点击顶部 **「保存配置」** 按钮，账号信息存入本地配置文件，下次启动自动加载。

> 只下载无需账号的传感器（DEM、SRTM、Landsat 系列）时，无需填写任何账号。

## 四、配置下载任务

### 4.1 上传 KML 区域文件

支持两种方式：

**方式 A — 拖拽上传**
将 `.kml`、`.ovkml`、`.kmz`、`.ovkmz` 文件拖入右栏上方的上传区域。上传成功后显示文件名和大小。

**方式 B — 手动输入路径**
点击「或手动输入路径 / 文件夹」，展开路径输入框，直接粘贴文件路径或文件夹路径：
- 输入单个文件路径：仅处理该文件
- 输入文件夹路径：批量处理目录下所有 KML 文件

### 4.2 选择传感器模式

传感器模块顶部提供两种模式切换：

**交付模式（默认）**：自动选中满足标准交付架构的核心传感器组合（SRTM + Sentinel-2 + Landsat + ASTER + ECOSTRESS + EMIT），适合大多数勘探场景。

**自定义模式**：手动点选任意传感器，完全自由搭配。

### 4.3 选择传感器

传感器按类型分为 7 个折叠分组，点击分组标题可展开/收起：

| 分组 | 传感器 |
|------|--------|
| 高程 | Copernicus DEM、SRTM |
| 多光谱 | Sentinel-2、Landsat 8/9、Landsat 7 ETM+、ASTER L1T、ASTER L2、MODIS、Landsat TIRS |
| 高光谱 | EMIT、Hyperion EO-1、AVIRIS-NG、EnMAP、PRISMA、DESIS、ZY-1 02D |
| 高分辨率 | SPOT 6/7、Pleiades、WorldView-2、WorldView-3 |
| 热红外 | ECOSTRESS |
| SAR 雷达 | Sentinel-1、OPERA RTC、ALOS PALSAR、ALOS-2 PALSAR-2 |
| 激光测高 | GEDI |

- 点击传感器卡片 = 选中（蓝色高亮）
- 再次点击 = 取消选中
- ⚠ 标记的传感器需要付费账号或无公开 API，点击时会显示说明

分组标题右侧会显示当前分组的已选/总数，例如「2/4 已选」。

### 4.4 数据预检（可选）

点击 **「数据预检」** 按钮，系统将对所有已选传感器在目标区域和时间范围内进行快速搜索（不下载），每个传感器卡片上显示可用景数。可用于快速筛选有效数据源，避免空跑。

### 4.5 设置时间范围

分别设置起始和结束日期（年 / 月 / 日三级下拉选择）。

> DEM 和 SRTM 不受时间范围限制。ALOS PALSAR 仅覆盖 2006–2011 年。

### 4.6 其他参数

| 参数 | 说明 | 推荐值 |
|------|------|--------|
| 云量 | 光学影像最大允许云量（%） | 20（山区可放宽至 30） |
| 最大景数 | 每个传感器最多下载的景数 | 5 |
| 裁剪到 KML 范围 | 自动将影像裁至研究区边界 | ✅ 建议开启 |
| 计算衍生产品 | 自动生成地表温度、温度梯度等 | ✅ 建议开启 |
| 打包标准交付目录 | 按标准文件架构整理结果 | ✅ 建议开启 |
| 输出目录 | 原始下载文件存放位置 | `./downloads` |
| 交付目录 | 打包后的标准交付结果位置 | `./delivery` |

## 五、开始下载

### 5.1 启动

点击右上角 **「开始下载」** 按钮。

任务启动后：
- 按钮变为 **「■ 停止」**
- 状态指示变为「运行中」（绿色圆点）
- 整体进度条出现，显示当前处理传感器和已下载景数
- 日志区域开始实时输出

### 5.2 读懂进度条

```
第 2/4 项：landsat          已下载 3 / 5 张    剩余约 12 分钟
████████████████░░░░░░░░░░░░░░░░░  45%
```

- **第 x/x 项**：当前处理第几个传感器（共选了几个）
- **已下载 x/x 张**：当前传感器已完成景数
- **剩余约 x 分钟**：基于近期下载速度估算的全任务剩余时间
- **进度条阶段**：
  - 0–89%：下载阶段
  - 93%：正在计算衍生产品
  - 97%：正在打包交付目录
  - 100%：全部完成

### 5.3 读懂日志颜色

| 颜色 | 含义 | 示例 |
|------|------|------|
| 绿色 | 成功完成 | `[完成] B04.tiff` |
| 黄色 | 警告，可继续 | `[警告] 裁剪失败，跳过` |
| 红色 | 错误 | `[错误] 认证失败` |
| 灰色 | 普通信息 | `找到 5 景，开始下载...` |
| 蓝色 | 命令行 | `$ python3 main.py ...` |

### 5.4 停止任务

点击 **「■ 停止」** 按钮立即终止当前任务。已下载的文件不会丢失，下次运行时已完成的文件自动跳过。

### 5.5 页面刷新后恢复

下载进行中刷新页面或重新打开 http://localhost:8080，系统会自动检测到任务仍在运行，并恢复显示完整的历史日志，无需重新启动。

## 七、代理与网络

下载启动时，系统会自动探测本地代理端口并按以下优先级尝试：

| 优先级 | 端口 | 协议 | 说明 |
|--------|------|------|------|
| 1 | 10793 | SOCKS5 | 快柠檬 SOCKS5（优先） |
| 2 | 10792 | HTTP | 快柠檬 HTTP（回退） |
| 3 | 7890 | HTTP | Clash |
| 4 | 1087 | HTTP | 其他代理工具 |
| 5 | 8080 | HTTP | 其他代理工具 |

检测到可用端口时，日志显示：`[代理] 自动探测到本地代理端口 10792，已启用`  
未检测到时，日志显示：`[代理] 未检测到本地代理端口`（若使用 TUN/虚拟网卡模式则无需配置）

**智能直连（no_proxy）：** 以下域名默认直连，不经代理，避免代理反而拖慢速度：
- `dataspace.copernicus.eu`（Sentinel-2）
- `eoweb.dlr.de`、`download.geoservice.dlr.de`（EnMAP / DESIS）
- `earthdata.nasa.gov`（NASA Earthdata）
- `usgs.gov`（Landsat）
- `jpl.nasa.gov`（AVIRIS-NG）

若需固定使用代理，在终端执行后再启动服务：
```bash
export https_proxy=http://127.0.0.1:10792
python3 web/app.py
```

## 八、常见问题

**Q：交付目录里没有生成 Word 报告**
需要安装 `python-docx`：`pip install python-docx`，然后重新运行打包即可。

**Q：启动失败提示"端口已占用"**
```bash
lsof -ti:8080 | xargs kill -9
python3 web/app.py
```

**Q：某个传感器一直显示"找到 0 景"**
- 检查时间范围是否在该传感器的覆盖时段内（ALOS PALSAR 仅 2006–2011 年）
- 检查研究区域是否在传感器覆盖范围内（GEDI 仅覆盖 51.6°S–51.6°N）
- 检查对应账号是否已填写并保存

**Q：下载途中断网，重新运行会重头下载吗**
不会。网络中断时下载器自动重试（最多 10 次，最长等待 120 秒）；若彻底中断后重新运行，已完成文件自动跳过，`.part` 临时文件从断点继续。

**Q：下载进度条卡住不动很久，没有任何输出**
连接可能出现"挂死"（代理半开连接）。系统内置 stall 保护：iter_content 超过 120 秒未收到新数据时自动断开并重试，正常情况下无需手动干预。若反复发生，可尝试切换代理节点或开启直连模式。

**Q：裁剪失败提示"Read failed"**
文件可能在上次下载时损坏（截断）。系统会在裁剪前自动检测并删除损坏文件、触发重新下载。若仍失败，手动删除对应的 `.TIF` / `.part` 文件后重新运行。

**Q：Sentinel-1 下载后裁剪失败提示"缺少坐标系信息"**
Sentinel-1 GRD 原始产品为斜距坐标，需先完成地理编码（Terrain Correction）。系统已集成 pyroSAR + ESA SNAP 自动处理，请确保已安装 SNAP（https://step.esa.int/main/download/snap-download/）。

**Q：Hyperion EO-1 显示警告**
Hyperion 数据已从 NASA 迁移至 USGS EarthExplorer，需手动下载后放入 `downloads/{区域名}/hyperion/` 目录。

**Q：EnMAP 下载卡在"等待订单"阶段**
EnMAP 采用订单制，DLR 处理需要时间（通常 10 分钟至数小时）。系统默认轮询 8 小时（`--order-timeout 28800`），超时后跳过打包，下次运行时自动恢复轮询。可增大超时值或次日再次运行。

**Q：EnMAP 提示 Playwright / Chromium 未安装**
```bash
pip install playwright
playwright install chromium
```

**Q：想只重新打包，不重新下载，怎么操作**
关闭「裁剪」开关和传感器选择，只保留「打包标准交付目录」开启，点击开始下载即可。

## 九、命令行使用（高级）

Web 界面底层调用的是 `main.py`，可直接通过命令行运行：

```bash
# 基础用法
python3 main.py \
  --kml ./area.kml \
  --sensor dem srtm sentinel2 landsat \
  --start 2024-01-01 \
  --end 2024-06-30

# 完整参数示例
python3 main.py \
  --kml ./kml_folder/ \           # 文件夹批量处理，多区域并发
  --sensor sentinel2 aster emit aster_l1t landsat_tirs alos2 \
  --start 2024-01-01 \
  --end 2024-12-31 \
  --cloud 20 \                    # 最大云量 20%
  --max-items 5 \                 # 每传感器最多 5 景
  --output ./downloads \
  --delivery-dir ./delivery \
  --area-workers 3 \              # 3 个区域并行处理
  --order-timeout 43200 \         # EnMAP 订单等待上限 12 小时
  --config config/credentials.yaml
```

| 参数 | 说明 |
|------|------|
| `--kml PATH` | 单个坐标文件或含多个坐标文件的文件夹（支持 .kml / .ovkml / .kmz / .ovkmz / .xlsx / .xls） |
| `--sensor` | 传感器名称，可多选（见下方完整列表） |
| `--start / --end` | 搜索时间范围（YYYY-MM-DD），DEM/SRTM 可省略 |
| `--cloud PCT` | 光学影像最大云量（0–100），默认 20 |
| `--max-items N` | 每区域每传感器最多下载景数，默认 5 |
| `--output DIR` | 原始下载目录，默认 `./downloads` |
| `--delivery-dir DIR` | 标准交付目录，默认 `./delivery` |
| `--no-clip` | 跳过裁剪，保留完整景 |
| `--no-derive` | 跳过衍生产品计算（地表温度/温度梯度/OTCI） |
| `--no-package` | 跳过打包到标准交付目录 |
| `--point-buffer DEG` | 点位 KML 的缓冲半径（度），默认 0.1°≈11km |
| `--area-workers N` | 多 KML 区域并发处理数，默认 2（设为 1 退化为串行） |
| `--order-timeout SEC` | EnMAP 等需下单传感器的订单轮询超时（秒），默认 28800（8 小时） |
| `--config FILE` | credentials.yaml 路径，默认 `config/credentials.yaml` |
